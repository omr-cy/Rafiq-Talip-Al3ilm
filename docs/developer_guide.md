# دليل المطور: إضافة آيات جديدة لقاعدة البيانات

هذا الدليل يشرح كيفية إضافة آيات وتفاسير جديدة لقاعدة بيانات التطبيق برمجياً.

## هيكل البيانات (Interface)

يتم تخزين الآيات في مخزن `ayahs` باستخدام الواجهة التالية:

```typescript
interface DailyAyah {
  id: string;
  text: string;
  surah: string;
  ayahNumber: number;
  tafsir: string;
  source?: string;
  createdAt: number;
}
```

## كيفية الإضافة

لإضافة آية جديدة، يجب استخدام كائن `db` المتاح في `@/src/lib/db`.

### مثال برمي:

يمكنك إضافة الكود التالي في ملف `db.ts` ضمن وظيفة تهيئة أو استدعاؤه من أي مكان آخر:

```typescript
import { db } from './db';

const addNewAyah = async () => {
  const newAyah = {
    id: `ayah_${Date.now()}`,
    text: "نص الآية الكريمة هنا",
    surah: "اسم السورة",
    ayahNumber: 1, // رقم الآية
    tafsir: "تفسير الآية هنا",
    source: "المصدر (اختياري)",
    createdAt: Date.now()
  };

  await db.ayahs.save(newAyah);
  console.log("تمت إضافة الآية بنجاح");
};
```

## التوصيات

1. **المعرف (ID)**: يفضل استخدام بادئة `ayah_` متبوعة بطابع زمني أو معرف فريد.
2. **النص**: يفضل كتابة نص الآية بالتشكيل الصحيح.
3. **التفسير**: يفضل استخدام تفاسير ميسرة وموثوقة.
4. **التحديث**: بعد إضافة آيات جديدة، قد يحتاج المستخدم لإعادة تحميل التطبيق لرؤية التغييرات إذا كانت الآية المختارة لليوم قد تم تحديدها بالفعل.
